# Product Documentation

## Problem Statement

Companies waste hundreds of hours monthly on manual meeting documentation.
Writing summaries, extracting action items, and sending follow-ups is
repetitive, error-prone, and delays team execution.

## User Personas

**Sarah - Team Lead**
- Runs 5+ meetings per week
- Spends 30 min after each meeting writing notes
- Needs action items tracked with clear owners

**Ahmed - Department Manager**
- Needs executive summaries of meetings he missed
- Wants to track decisions and risks across teams

## User Journey

1. Sign in to the app
2. Create a new meeting (title, date, participants, department, type)
3. Upload the transcript file (.txt)
4. AI analyzes and returns structured output
5. User reviews and edits the results
6. User exports PDF or sends follow-up email

## MVP Scope

- Auth (sign in / sign up)
- Create meeting + upload transcript
- AI analysis with all required fields
- Edit results inline
- Export as PDF
- Generate follow-up email
- Dashboard with meeting history
- Basic search

## Future Roadmap

- Audio file transcription (Whisper API)
- Calendar integration
- Email notifications
- Role-based access control
- Analytics dashboard
- Chat with previous meetings
